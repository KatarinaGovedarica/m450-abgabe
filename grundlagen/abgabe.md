# Aufgabe 1

- **Unit Tests**:  hier testet man isolierten source code um sicherzustellen das es wie erwartet funktioniert.
- **Component Tests**:  bei component tests, testet man einzelne Komponente anstatt eine ganze Applikation oder Seite. z.B. man testet nur das fotokarussel auf der Page.
- **Blackbox Tests**:  beim blackbox testing testet man, ohne zu wissen was der code ist. das hilft einfach das system von einer benutzerseite aus zu testen.

# Aufgabe 2

- **SW-Fehler**:

z.B. ein taschenrechner-pogramm rechnet 2 + 2 = 5 anstatt 2 + 2 = 4. 

Ein SW Fehler erfüllt die Anforderung nicht.

- **SW-Mangel**:

z.B.  die ladezeit einer Seite ist 8 Sekunden lang, gefordert wurden 2 Sekunden

Ein SW Mangel erfüllt die Anforderung nicht angemessen.
